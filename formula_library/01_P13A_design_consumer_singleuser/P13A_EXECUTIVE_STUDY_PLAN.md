### BUCKET 1: Physical Product Design
**Questions:** ~15 | **Priority:** ðŸŸ¢ GREEN (Master this)

**Board Slide Bullets:**
- **What:** "Design a better X" or "Design X for Y" - core product design framework
- **Framework:** User Context â†’ Pain Points â†’ Core Design â†’ Tradeoffs â†’ Risks
- **User Context:** Who is the user? (Demographics, physical/mental capabilities, usage environment), What is their current experience? (Existing solutions, pain points)
- **Pain Points:** What problems need solving? (Functional problems, usability issues, accessibility barriers), Prioritize pain points (Frequency, Severity)
- **Core Design:** Key features (Must-have functionality, usability improvements, accessibility features), Design principles (Simplicity, Safety, Durability)
- **Tradeoffs:** Cost vs quality, Features vs simplicity, Customization vs standardization
- **Risks:** Safety risks, Adoption barriers, Manufacturing constraints

**Concrete Examples:**
- "Design better alarm clock: User Context (adults who need reliable wake-up, current alarms are jarring or unreliable), Pain Points (hard to set, too loud/quiet, snooze confusion), Core Design (simple interface, gradual volume, smart snooze), Tradeoffs (features vs simplicity), Risks (safety - oversleeping, adoption - learning curve)"
- "Design elevator for blind users: User Context (blind users, need to navigate independently, current elevators rely on visual cues), Pain Points (can't see floor numbers, can't see buttons, don't know when doors open), Core Design (braille buttons, audio announcements, tactile feedback), Tradeoffs (cost vs accessibility), Risks (safety - getting lost, adoption - training needed)"

- Q276: Design a chair for the disabled. (accessibility design angle)
- Q381: Design a next-generation elevator. (innovation angle)

### BUCKET 2: Accessibility & User Context
**Questions:** ~5 | **Priority:** ðŸŸ¡ YELLOW (High-yield but needs practice)

**Board Slide Bullets:**
- **What:** "Design X for [specific user group]" - accessibility and user-specific design
- **Approach:** Same framework, with focus on accessibility and user constraints
- **Accessibility Design:** Understand user limitations (Physical, mental, sensory), Design for inclusivity (Universal design principles, adaptive features), Test with users (User testing, feedback, iteration)
- **User Context:** Specific user needs (Children, elderly, disabled, specific demographics), Usage environment (Home, public, workplace), Constraints (Physical limitations, cognitive limitations, environmental constraints)

**Concrete Examples:**
- "Design for children: User Context (children have smaller hands, less strength, need safety, used at home/school), Pain Points (too complex, too heavy, safety concerns, not engaging), Core Design (simple interface, lightweight, safety features, engaging design), Tradeoffs (features vs simplicity - keep simple, Cost vs safety - prioritize safety), Risks (Safety - injuries, Adoption - age-appropriate)"
- "Design for elderly: User Context (elderly have reduced vision/hearing, reduced dexterity, need simplicity, used at home), Pain Points (small text, complex controls, hard to see/hear, physical limitations), Core Design (large text/buttons, simple interface, audio feedback, easy grip, high contrast), Tradeoffs (features vs simplicity - prioritize simplicity, Cost vs accessibility - worth the cost), Risks (Safety - confusion, Adoption - learning curve)"

**Representative Questions (Do 5 only):**
- Q605: Design an elevator for blind users. (accessibility design angle)
- Q268: Design a bookshelf for elderly people. (elderly-specific design angle)
- Q278: Design a clock for children. (children-specific design angle)
- Q564: Design an alarm clock for children 100 years in the future. (futuristic + children angle)
- Q267: Design a bookshelf for children. (children-specific design angle)

**â¤ï¸ Reusable Narrative (Base Story - Adapt for Each Question):**
