## üéØ How to Use

### For Each Role:

1. **Understand the 4 pillars** - These are what get you fired
2. **Map questions to pillars** - Every question tests one of these
3. **Know the thresholds** - Safe vs Warning vs Fucked
4. **Understand the impact** - What happens when things break

### Example:

**Question**: "How do you handle duplicate records?"

**Pillar**: [1] DUPLICATION

**Intent**: Test if you understand data quality fundamentals

**Your Answer Should:**
- Show you understand business impact (revenue inflation)
- Demonstrate prevention strategy (idempotency)
- Know thresholds (< 2% safe, > 5% fucked)
